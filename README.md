# NEON DRIFT · 霓虹幸存者

一款浏览器端的 **roguelite 生存射击小游戏**，纯原生 HTML5 + Canvas + ES Modules 实现，
**零第三方依赖、零美术/音频资源文件**（所有视觉用 Canvas 程序绘制，所有音效用 Web Audio API 实时合成）。

霓虹故障（neon / glitch）美术风格：发光几何体、扫描线滤镜、屏幕暗角、粒子拖尾与屏幕震动。

---

## 玩法

你驾驶一艘霓虹飞船，在不断涌来的几何敌群中生存。武器**全自动开火**，你只需专注走位、吃经验、选强化。

- **移动**：`W A S D` 或 方向键（移动端：在画面任意处按下拖动，即出现浮动虚拟摇杆）
- **暂停**：`P` 或 `ESC`
- **升级三选一**：升级时鼠标点击卡片，或按 `1` / `2` / `3`

### 核心循环
击杀敌人 → 掉落经验晶体 → 升级 → 三选一强化武器/属性 → 面对更强的敌潮与 Boss。

### 局外永久成长（让每局不再孤立）
每局结束依据「击杀数 + 存活时长 + 击败母核数」结算 **暗物质核心 ◆**（自动存档于浏览器本地）。
在主菜单的 **强化实验室** 中花费核心购买**跨局永久加成**，共 12 项、等级更高、花费曲线更陡：
强化装甲 / 武器校准 / 引擎调校 / 纳米基质 / 引力核心 / 贪婪协议，以及
**暴击矩阵 / 时序压缩(攻速) / 相位护盾(减伤) / 学习协议(经验) / 战术预载(开局免费加点) / 应急重构(阵亡复活)**。
> 平衡：全部买满需累积数千核心，远超单局产出，因此**不可能一局就买满全部加点**，长期养成空间充足。
这些加成在每局开局自动生效——**即使失败也在变强**，形成长期游玩驱动。主菜单还会展示历史最佳记录。

### 机库 · 战机外观与抽卡（棱牌 ✦）
主菜单进入 **机库**，可自由更换战机外观，每种外观都有**独立造型**与**专属特性**：
- **独立货币「棱牌 ✦」**：每局依据击杀 / 击败母核 / 存活时长结算，与强化用的暗物质核心互不挤占。
- **稀有度**：普通 / 稀有 / 史诗 / 传说，决定抽卡权重与特性强度。
- **星级成长（★1–★5）**：重复抽到同一外观会**升星**，专属特性随星级线性增强；满星后重复抽到则**返还棱牌**。
- **抽卡**：单抽或十连（打折），按稀有度权重随机产出，结果界面逐张翻开展示「NEW / 升星 / 返还」。

| 外观 | 稀有度 | 专属特性（随星级增强） |
|------|--------|------------------------|
| 漂移者 | 普通 | 移动速度提升 |
| 余烬 | 普通 | 全武器伤害提升 |
| 壁垒 | 稀有 | 受到伤害减免 |
| 彗核 | 稀有 | 移速 + 拾取范围 |
| 幻影 | 史诗 | 暴击率提升 |
| 涡轮 | 史诗 | 武器攻速提升 |
| 新星君主 | 传说 | 全伤害 + 起始生命 |
| 奇点 | 传说 | 暴击 + 攻速 + 击杀吸血 |

> 选中的外观特性在开局注入，与永久强化、局内升级三重叠加。

### 图鉴 · 情报库与收集度奖励
主菜单进入 **图鉴**，查看已遭遇的敌人 / Boss / 武器 / 道具：
- **黑色轮廓 + 问号**：尚未发掘的目标。
- **主题色符号 + 名称**：局内首次遭遇即录入图鉴（Boss/普通敌人分区显示）。
- **收集度进度条** 显示总收录 / 总数，四档 **里程碑奖励**：
  - `初识` 3 项 → 100 棱牌 ✦
  - `见闻录` 7 项 → 300 棱牌 ✦
  - `深度勘测` 12 项 → 500 暗物质核心 ◆
  - `万象编纂` 全收集 → 解锁传说皮肤 **万象编纂**（含主动技能「归零协议」）
- **主动技能「归零协议」**：装备万象编纂皮肤后，桌面端按 `空格` / 移动端点右下角圆钮释放：
  对屏内敌人造成范围重创、清空敌方弹幕、玩家短暂无敌，30s 冷却（受局内攻速强化影响）。

### 武器（可通过升级解锁 / 进阶）
| 武器 | 说明 |
|------|------|
| ✦ 脉冲枪 | 自动瞄准最近敌人，可升级散射 / 射速 / 穿透 |
| ◓ 轨道核 | 环绕飞船的光球，碰触即伤害 |
| ❂ 灼蚀场 | 身周持续范围灼烧 |
| ✸ 超新星 | 周期性环形弹幕爆发 |
| ⚡ 电弧链 | 周期性放出电弧，在最近的多个敌人间跳跃 |

局内被动还包含：**精准校准(暴击) / 超频循环(攻速) / 噬能装甲(击杀吸血)** 等，配合局外强化叠加成长。

### 敌人
游荡者（三角）、突袭体（高速摆动）、壁垒（高血量）、裂解体（死亡分裂），
以及每 2 分钟降临、**分三种阶段化变体轮换**的 Boss（每轮血量递增）：
- **母核**（稳健）：环形弹幕 / 扇形散射 / 召唤突袭体。
- **裂能体**（敏捷）：双臂反向螺旋 / 突进冲刺+紧密扇射 / 高速环爆。
- **湮灭者**（沉重）：双层同心环波 / 宽域横扫 / 召唤壁垒+裂解体。
每种 Boss 造型、攻击套路与出招节奏各异，出招前有橙色蓄力预警环，需要主动走位躲避。

### 掉落道具
经验晶体、回血、磁吸（吸附全场经验）、湮灭炸弹（对玩家周围一定**范围内**的敌人造成毁灭性打击，范围外免疫，拾取时会显示冲击波环提示波及区域）。

## 即开即玩（推荐）

项目已内置零依赖打包脚本，可生成**完全自包含的单文件 HTML**，无需服务器、双击即玩：

```bash
npm run build          # 或： node build.mjs
```

构建产物在 `dist/` 目录：

- **`dist/neon-drift.html`** —— 单个文件，CSS 与全部 JS 已内联。
  - **电脑**：直接双击用浏览器打开即可游玩（`file://` 协议无障碍）。
  - **手机**：把这个文件传到手机（微信/AirDrop/网盘等），用浏览器打开即可；或托管到任意静态空间，扫码访问。
  - 存档（暗物质核心、永久升级、最佳记录、机库外观）保存在浏览器本地，无需联网。
  - **续玩快照**：进行中的对局会自动保存到本地（每 4 秒 / 暂停 / 关闭页面时），下次打开主菜单出现 **继续上局**，从上次的等级、装备与难度进度接着玩，无需从头开始。
- **`dist/index.html` + `dist/game.js`** —— 同等内容的分离版，双击 `index.html` 同样可玩。

> 提示：本游戏自适应桌面与移动端。桌面用 `WASD`/方向键；移动端在画面任意处按下拖动即出现虚拟摇杆，武器自动开火。

## 开发方式（源码调试）

源码使用 ES Modules（`import`/`export`），开发时需通过 **HTTP 服务器**打开（`file://` 会拦截模块加载）：

```bash
# 方式一：Python 3（macOS 自带）
python3 -m http.server 8000      # 或： npm run dev

# 方式二：Node.js
npx serve .
```

然后浏览器访问 <http://localhost:8000> 即可游玩源码版（改动即时生效，无需打包）。

## 项目架构

采用分层 + 对象池设计，关注点分离、易于扩展与维护：

```
neon-survivors/
├── index.html              # 入口页面 + Canvas + CSS 滤镜叠层
├── css/
│   └── style.css           # 霓虹故障风格样式 / DOM 界面
└── js/
    ├── main.js             # 应用入口：装配依赖、启动循环
    ├── config.js           # 集中式配置（数值平衡单一数据源）+ 状态枚举
    ├── utils/              # 通用工具
    │   ├── Vector2.js      #   二维向量
    │   └── math.js         #   数学 / 随机 / 格式化
    ├── core/               # 引擎核心（与具体玩法无关）
    │   ├── Game.js         #   核心控制器 / 中介者 / 主循环 / 状态机
    │   ├── Input.js        #   键盘输入管理
    │   ├── AudioFx.js      #   Web Audio 程序化音效
    │   ├── Camera.js       #   跟随摄像机 + 屏幕震动 + 坐标换算
    │   ├── Pool.js         #   通用对象池（活跃列表 + 空闲栈）
    │   ├── SpatialGrid.js  #   空间网格哈希（近邻碰撞查询）
    │   ├── GlowCache.js    #   发光精灵缓存（替代 shadowBlur）
    │   ├── SaveData.js     #   本地存档持久化（localStorage）
    │   └── ParticleSystem.js  # 粒子 / 浮动文字（对象池）
    ├── entities/           # 游戏实体
    │   ├── Player.js       #   玩家 + 武器系统
    │   ├── Enemy.js        #   多类型敌人
    │   ├── Projectile.js   #   抛射物
    │   └── XPGem.js        #   经验晶体 / 道具（含超时消失）
    ├── systems/            # 游戏系统（作用于实体集合）
    │   ├── SpawnSystem.js      # 生成 / 难度爬升 / Boss
    │   ├── CollisionSystem.js  # 碰撞检测与结算（走空间网格）
    │   ├── UpgradeSystem.js    # Roguelite 局内升级抽卡
    │   ├── MetaProgression.js  # 局外永久成长（货币 / 永久升级）
    │   ├── Codex.js            # 图鉴：局内发现 / 里程碑奖励（棱牌 / 核心 / 皮肤 / 主动技能）
    │   └── Skins.js            # 局外机库：外观 / 抽卡 / 星级成长 / 专属特性 / 造型绘制
    └── ui/                 # 表现层
        ├── HUD.js          #   Canvas 内抬头显示（含 Boss 血条）
        └── Screens.js      #   DOM 覆盖界面（菜单/商店/机库抽卡/升级/暂停/结算）
```

### 设计要点
- **单一数据源**：所有可调数值集中在 `config.js`，平衡性调整只改一处。
- **实体 / 系统解耦**：实体不直接依赖彼此，而是通过 `Game` 暴露的服务方法
  （`spawnProjectile` / `damageEnemy` / `findNearestEnemy` 等）交互（中介者模式）。
- **对象池**：敌人、子弹、粒子、掉落物均复用对象，避免运行时频繁 GC 造成卡顿。
- **视图分离**：高频游戏画面用 Canvas 绘制；可交互的菜单/卡片用 DOM，各取所长。
- **状态机**：`MENU / SHOP / HANGAR / CODEX / PLAYING / LEVELUP / PAUSED / GAMEOVER` 清晰驱动流程。

## 自定义与扩展
- 想加新武器：在 `config.js` 增加配置 → 在 `Player.js` 添加触发逻辑 → 在 `UpgradeSystem.js` 增加升级项。
- 想加新敌人：在 `config.enemies` 增加定义 → 在 `Enemy.js` 的 `render` 增加造型 → 在 `SpawnSystem._enemyPool` 纳入权重。
- 想调整难度：修改 `config.spawn` 与 `config.progression`。
- 想加新外观：在 `Skins.js` 的 `SKINS` 表增加条目（`shape` / `rarity` / `perk` / `perkText`），并在 `drawShip()` 补充对应 `shape` 造型分支即可。
